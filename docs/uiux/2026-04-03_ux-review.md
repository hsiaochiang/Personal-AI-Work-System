# UX Review 2026-04-03

## Review Scope

- 驗收 `chatgpt-adapter` 對 `/extract` 使用流程的最小 UX 變更
- 檢查重點：
  - 既有 textarea 仍是唯一主輸入面
  - 上傳 JSON/TXT 後不需要跳到新的 flow 或頁面
  - plain text fallback 仍可直接使用

## Findings

- `/extract` 沒有被擴大成多來源 selector；使用者仍只面對單一輸入區與單一「提取候選知識」主按鈕。
- 檔案上傳只是把內容載入既有 textarea，沒有新增第二條 extraction path，降低使用者分流成本。
- 使用者在檔案載入後若直接修改 textarea，狀態會切回手動貼上模式，避免沿用過期的 JSON hint。
- ChatGPT JSON 多 conversation 的限制目前仍存在，但已用 deterministic 規則與文案說明控制風險；完整挑選留給後續 change。

## Decision

- UX review PASS
- 目前流程維持單一路徑與低摩擦，符合本 change 的最小安全範圍
