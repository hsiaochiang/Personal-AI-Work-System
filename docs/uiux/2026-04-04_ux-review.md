# UX Review 2026-04-04

## Review Scope

- 驗收 `import-ui-multi-source` 對 `/extract` 操作流程的 UX 影響
- 檢查重點：
  - 使用者是否更容易理解「現在要用哪個匯入入口」
  - 切換來源時，舊狀態是否會被清理，避免錯用前一個模式的資料
  - 既有單一 textarea + 單一「提取候選知識」節奏是否仍被保留

## Findings

- selector 讓使用者先決定來源，再看到對應控制，明確降低了原本「Copilot block / ChatGPT upload / textarea 同時出現」的判讀負擔。
- 使用者切換來源時，前一個模式的 `selectedImportFile` / `selectedConversationDoc` / `selectedCopilotSession` 會被清掉，避免用錯資料繼續提取。
- ChatGPT 模式不再默默 fallback 到 plain text；若輸入不符合 ChatGPT 內容，使用者必須明確切回 `plain`，這和 selector 的心智模型一致。
- 雖然入口更明確，但主流程仍維持單一 textarea 預覽與單一「提取候選知識」按鈕，沒有引入第二套 extraction path。

## Decision

- UX review PASS
- 本次流程比前一版更可預期，且沒有增加額外操作成本；適合作為 V3 Import UI 的收尾形態
