# [INBOX] Review Gate 未要求多步驟流程端到端審查

日期：2026-04-10  
來源：observing  
類型：product-idea  
狀態：已納入版本（2026-04-10 補充至 35-quality-gate.md）

---

## 問題或動機

在調查 `/extract` UX 問題的過程中，發現 2026-04-04 的 UI/UX review 文件存在且評定為 PASS，但該 review 只涵蓋了被修改的 Step 1（來源 selector），沒有審查使用者接下來操作的 Step 2（候選審核）。

結論：**現有 Done Gate 規則沒有要求「受影響的完整使用路徑都要被審查」**，只要求「直接修改的部分要被審查」。

這會導致開發者在多步驟流程中只審查自己改的那步，而忽略下游步驟是否也因為這次改動而受到影響。

## 期望體驗

Done Gate 規則補充：凡修改多步驟流程中任何一步，UI/UX review scope 必須覆蓋**完整使用路徑**（從入口到最終行動按鈕），不得只審查被直接修改的步驟。

## 建議方向

補充一條規則到 `35-quality-gate.md`。

## 優先序評估

- 影響範圍：high（防止同類問題重複發生）
- 實作複雜度：low（文件修改）
- 阻塞現有流程：no（但不修代表同類問題會持續出現）
