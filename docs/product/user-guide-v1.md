# 個人 AI 工作台 — 使用者操作手冊

> 這份文件描述 V1 完成後，你每天怎麼用這個網站。
> 每個場景都用「你做什麼 → 系統給你什麼」的格式說明。

## 啟動方式

```bash
cd D:\program\Personal-AI-Work-System\web
npm start
```

瀏覽器開啟 http://localhost:3000，進入工作台。

---

## 日常使用情境

### 情境 1：早上開始工作 — 快速掌握全局

**你的需求：** 「我有 3 個專案，今天該做什麼？」

**操作步驟：**

1. 打開瀏覽器 → 看到 **Projects Hub**
2. 一眼看到 3 個專案的卡片，各自顯示：
   - 專案名稱、目前 Phase、最近更新時間
   - 狀態色標（綠色 = 正常推進、黃色 = 有注意事項、紅色 = 有 Blocker）
3. 點擊你想進入的專案

**畫面你會看到：**

```
┌─────────────────────────────────────────────────────────┐
│ 儀表板   工作區   分析   治理          🔍 搜尋...  🔔 ⚙ │
├──────────┬──────────────────────────────────────────────┤
│          │                                              │
│ 專案列表  │  📊 專案概覽                                 │
│ 路線圖    │                                              │
│ 驗證      │  ┌────────┐ ┌────────┐ ┌────────┐ ┌───────┐ │
│ 決策日誌  │  │目前階段 │ │ 閘門   │ │ 阻礙項 │ │ 驗證  │ │
│          │  │Phase 2  │ │ PASS   │ │  0     │ │ PASS  │ │
│          │  └────────┘ └────────┘ └────────┘ └───────┘ │
│          │                                              │
│          │  Phase 進度表                                  │
```

### 情境 2：查看某個專案的詳細狀態

**你的需求：** 「Phase 2 做到哪了？有什麼在卡住的？」

**操作步驟：**

1. 在側邊欄點擊「專案列表」
2. 點擊某個 Phase（例如 Phase 2）
3. 進入 **Detail 頁**

**畫面你會看到：**

- **驗收準則進度**（AC1-AC5 各有完成百分比進度條）
- **Blocker 卡片**（紅色區塊，列出待處理的阻礙項）
- **Decision Log**（這個 Phase 做過哪些決策）
- **Runlog Timeline**（最近的操作時間軸）
- **系統狀態摘要**（專案權限、資料同步、最後變更時間）

**你可以做的事：**

- 瀏覽所有 AC 的進度，確認哪些未完成
- 看 Blocker，決定要不要先處理
- 讀 Decision Log，確認之前為什麼這樣決定

---

### 情境 3：審核記憶候選

**你的需求：** 「AI 對話中產生了一些有用的知識，我要決定哪些值得留下。」

**操作步驟：**

1. 頂部導覽點「治理」
2. 進入 **Memory Review** 頁
3. 左側分類欄選擇類別（偏好規則 / 輸出模式 / 決策紀錄 / 專案背景 / 技能候選 / 任務模式）
4. 逐條審核候選

**每張候選卡片顯示：**

```
┌──────────────────────────────────────────────────┐
│ 🔤 一律使用正體中文回應                    🏷 偏好規則 │
│ ● 偏好規則  📅 更新於 2026-03-25  📈 使用次數: 4,120 │
│                                                    │
│ 無論輸入來源為何，回應必須使用台灣習慣的正體中文       │
│ (Traditional Chinese, Taiwan)。避免使用中國大陸         │
│ 術語（例如：質量→品質、水平→水準、支持→支援）。        │
│                                                    │
│               [ ✅ 採用 ]    [ ❌ 拒絕 ]             │
└──────────────────────────────────────────────────┘
```

**你可以做的事：**

| 操作 | 結果 |
|------|------|
| 點「採用」 | 候選寫入對應的 `docs/memory/*.md` |
| 點「拒絕」 | 系統要求輸入拒絕原因，候選標記為已拒絕 |
| 看「衝突」卡片（紅色邊框） | 新候選與既有規則矛盾，你決定保留哪個 |
| 點分類標籤 | 切換到不同類別的候選清單 |

**審核完後：**

- 被採用的知識自動寫入對應的記憶檔案
- 下次 AI 對話時，這些知識已經在你的記憶庫中
- 你的 AI 工作夥伴會越來越懂你

### 情境 4：追蹤決策

**你的需求：** 「之前好像決定過不用框架、用 vanilla JS，讓我確認一下。」

**操作步驟：**

1. 頂部導覽點「治理」→ 側邊欄點「決策日誌」
2. 或使用頂部搜尋列，輸入「vanilla JS」
3. 在 **Decisions** 頁看到所有決策的清單

**每條決策顯示：**

- 決策內容（例如：「先靜態 HTML 再評估框架」）
- 決策理由
- 影響範圍
- 決策日期
- 來源（哪次對話、哪個 Phase）

**你可以做的事：**

- 瀏覽所有決策，確認方向一致
- 用搜尋找特定決策
- 用標籤篩選（技術路徑 / 開發原則 / 輸出限制 / ...）

---

### 情境 5：切換專案

**你的需求：** 「我有另一個專案也要看進度。」

**操作步驟：**

1. 點頂部左側回到 Projects Hub
2. 點擊另一個專案卡片
3. 整個工作台的內容切換到該專案

**系統行為：**

- Overview → 顯示該專案的 roadmap
- Memory → 顯示該專案的記憶檔
- Decisions → 顯示該專案的決策
- 側邊欄標題顯示目前專案名稱

**設定新專案：**

1. 在 Projects Hub 點「+ New Project」
2. 選擇本地資料夾路徑
3. 系統自動建立標準資料夾結構（`docs/`, `docs/memory/`, `docs/handoff/`, ...）

---

### 情境 6：從對話提取知識（Phase 5 完成後）

**你的需求：** 「剛跟 Copilot 討論完架構，有些決策和偏好值得留下來。」

**操作步驟：**

1. 頂部導覽點「分析」
2. 進入 **提取引擎** 頁
3. 有兩種匯入方式：
   - **貼上文字**：直接複製對話紀錄貼上
   - **匯入檔案**：上傳對話匯出的 JSON/MD 檔
4. 系統自動解析，產生候選清單

**系統會做的事：**

- 識別對話中的偏好（「使用者偏好 X」）
- 識別決策（「最後決定用 Y」）
- 識別模式（「這是第 3 次用這個做法」）
- 各候選附上信心評分（高/中/低）
- 與既有記憶比對去重

**然後你在 Memory Review 審核這些候選（同情境 3）。**

---

### 情境 7：開始新的 AI 對話（日常工作流的核心）

**你的需求：** 「要跟 Copilot/Codex 開新對話繼續做事，不想重新交代背景。」

**操作步驟：**

1. 在工作台查看當前任務頁（目前已有）
2. 複製 handoff 內容
3. 貼進新 AI 對話的第一則訊息

**你得到的效果：**

- AI 立刻知道專案目標、當前進度、已定案決策
- 不用花 5 分鐘重新交代背景
- 降低 AI 理解偏差的機率
- 每次新對話的起點越來越高（因為記憶持續累積）

## 頁面總覽

| 頁面 | 頂部 tab | 功能 | 對應 Phase |
|------|---------|------|-----------|
| Projects Hub | 儀表板 | 多專案卡片 + 聚合狀態 | Phase 4 |
| Overview | 儀表板 | 單專案 KPI + Phase 進度 + Change 表 | Phase 2 |
| Detail | 工作區 | AC 進度 + Blocker + Decision Log + Runlog | Phase 2 |
| Memory Review | 治理 | 記憶審核（採用/拒絕/衝突） | Phase 3 |
| Decisions | 治理 | 決策清單 + 搜尋 + 篩選 | Phase 3 |
| Rules | 治理 | 偏好/模式/任務 集中檢視 | Phase 3 |
| Extraction | 分析 | 對話貼上 → 候選知識提取 | Phase 5 |
| Current Task | 工作區 | 當前任務交接資訊（已有） | Phase 1 ✅ |

## 搜尋功能

全域搜尋列位於頂部導覽右側，支援：

- 搜尋 roadmap 內容
- 搜尋 memory 條目
- 搜尋 decision log
- 搜尋 runlog
- 結果即時顯示，點擊跳轉到對應頁面

## 你不需要做的事

| 以前要做的 | 現在系統幫你做 |
|----------|-------------|
| 手動翻 docs 資料夾找檔案 | 全部在 UI 瀏覽 |
| 記住哪個決策在哪串對話裡 | Decisions 頁集中查看 |
| 手動整理對話裡的有用知識 | 提取引擎半自動產生候選 |
| 每次新對話重新交代背景 | 複製 handoff 貼上即可 |
| 記住多個專案各自的狀態 | Projects Hub 一眼看完 |
| 擔心好的做法沒留下來 | 記憶審核讓你主動決定什麼值得留 |

## 技術注意事項

- **純本地運行**：所有資料都在你的電腦上，不上傳任何雲端
- **資料來源是 markdown**：你隨時可以用任何編輯器修改原始檔案
- **即時反映**：修改 markdown 後，頁面自動（或重整後）更新
- **不需要安裝額外軟體**：只需要 Node.js（你已經有了）
