- **交付：** `web/public/extract.html` + `web/public/js/extract.js`
- 3 步驟流程：貼文字 → 審核候選 → 寫回記憶

## 關鍵決策

| 決策 | 理由 |
|------|------|
| 技術棧維持純 HTML + vanilla JS，不引入框架 | 減少前置設定摩擦，保持可直接執行，符合「最小安全修改」原則 |
| Handoff Builder 範本結構硬編碼在前端 JS | 範本格式穩定，不需要每次讀取檔案；若需客製化再改為後端讀取 |
| 提取引擎採啟發式 regex，不用 NLP | Phase 3 目標是閉環可用，而非提取精確；NLP 留待 V2 |
| 衝突偵測採否定詞前綴比對 | 簡單可執行，覆蓋常見衝突模式；複雜語意比對留待 V2 |
| 多專案切換只做 localStorage + 顯示，不做資料來源隔離 | V1 使用者只有一個專案；資料來源隔離留待 V2 |
| roadmap 從「UI polish 優先」改為「閉環優先」 | 依據 `2026-03-28_project-assessment-and-recommendations.md`；核心閉環是最重要的價值 |

## 技術選型紀錄

| 元件 | 選型 | 原因 |
|------|------|------|
| Server | Node.js HTTP（內建模組，無額外 dependency） | 零安裝負擔，已有基礎 |
| 樣式 | Slate Nexus Design System（自訂 CSS variables） | 已建立，保持一致性 |
| 字型 | Manrope（headline）+ Inter（body）via Google Fonts | 已有，不改 |
| Icons | Material Symbols Outlined | 已有，不改 |
| 搜尋 | 前端字串 includes（無搜尋 library） | 資料量小，不需 Fuse.js 等 |
| 儲存 | localStorage（專案選擇）+ 直接讀寫 markdown 檔案 | 符合現有架構 |
