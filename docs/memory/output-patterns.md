# Output Patterns

## 使用原則

- 只記錄跨多次任務仍有價值的交付樣式
- 以結構、語氣、詳略程度為主，不存單篇內容本身
- 格式應易讀、易掃描、可快速重用

## 已確認的輸出模式

### 輸出模式 1：先收斂概念，再提出下一步

- 適用情境：規劃、設計、策略討論
- 結構特徵：
  - 先整理當前結論
  - 再說明其意義
  - 最後提出最自然的下一步
- 語氣偏好：
  - 白話
  - 清楚
  - 不過度術語化
- 詳略程度：
  - 要足夠支撐判斷
  - 但不要堆砌不必要的分類
- 狀態：已確認

### 輸出模式 2：定案後同步寫入 markdown

- 適用情境：規劃、規則、決策確認後
- 結構特徵：
  - 先討論
  - 確認定案
  - 再同步寫進對應檔案
- 目的：
  - 讓對話結果轉成可持續使用的專案記憶
- 狀態：已確認
